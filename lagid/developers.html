Technical Specifications V1.4

Accepting Diesels (Ð)

Third-party applications (stores, bots, websites) can accept Diesels by processing .diesel files or raw signed payloads. Integration requires the tweetnacl library.

1. Payload Structure

A transaction payload is a Base64 encoded JSON object containing the following mandatory fields:

{
  "type": "BEARER",
  "amount": 100.00,
  "issuer": "node_...",
  "public_key": "...", // Ed25519 Public Key
  "tx_count": 5,
  "timestamp": 1776911254000,
  "expiresAt": 1776911426000,
  "signature": "...", // Detached Ed25519 Signature
  "serial": "Ð-TX-..."
}

2. Verification Algorithm

To accept a payment, your server or app must perform these checks:

  • Cryptographic: Verify the signature against the public_key and the JSON payload (excluding signature/serial).
  • Identity Binding: Ensure the issuer ID is the correct Iroh Node ID derived from the public_key.
  • Expiration: Reject payloads where Date.now() > expiresAt.
  • Integrity: Re-calculate the serial using the pipe-delimited standard: signature|issuer|recipient|amount|tx_count|timestamp.

Swarm Infrastructure

Running a Node

LAG-ID operates on top of the Iroh Network. To support the community network, you can run an Iroh Relay or a Persistence Node.

  • Community Namespace: All P2P operations require a specific cryptographic Namespace ID. The official Mainnet Namespace will be announced upon Swarm activation.
  • Relay Address: Standard nodes should connect to the official relays defined in the network_config.json for discovery.
  • Persistence: Nodes acting as "Paradores" must implement the append-only Nullifier Ledger API to synchronize transaction finality.

Root of Trust

Official Prize Verification

If you are implementing a system that handles "Official Prizes", you must hardcode the Master Authority Key to verify the origin of MINT assets.

const ADMIN_PUBKEY = 'gn4OzrxzOR436jeB8M0d6dDcVHCkKXi0R4NIpc/10yw=';

Any asset with issuer: "LAG-ID_FORGE" must be rejected if its public_key does not match the one above.
